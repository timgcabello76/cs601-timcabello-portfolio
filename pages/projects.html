Course Projects

// CS601 — Frontend Web Application Development, Summer 2025 — Prof. Christian Hur

Six assignments across six weeks. Each one introduced something I either hadn't used before or had only touched in a limited way. Notes below are honest about what was new and what gave me trouble.

HW1 Module 1

Interactive Web Form

HTML5 CSS3 JavaScript

A subscription sign-up form with all validation written in JavaScript. No HTML5 required attributes, no frameworks. Covers name fields, email format, package selection, and a confirmation checkbox. On a successful submit the form hides and shows a summary. Errors are written to the DOM rather than using alerts.

Notes Used arrow functions and array methods for the validation logic as the assignment required. Getting the error clearing to behave right when someone edits a field after failing validation took more debugging than I expected. Writing errors to the DOM instead of alerts ended up looking a lot cleaner.
HW2 Module 2

Async Drag-and-Drop App

HTML5 CSS3 JavaScript Fetch API Drag & Drop API

A fruit and vegetable sorting app that fetches JSON data asynchronously, renders a mixed list of draggable items, and lets users sort them into two labeled drop zones by category. JSON hosted on jsonbin.io. Uses the native HTML5 drag-and-drop API, no libraries.

Notes Fetch with async/await was fairly straightforward. The drag-and-drop API was not. Getting dragover and drop events to fire reliably took a lot of testing. Tagging each element with its category early on turned out to be the thing that made the drop zone validation actually work.
HW3 Module 3

CSS Portfolio Page

HTML5 CSS3 Flexbox CSS Grid

A single-page portfolio layout using CSS Grid for the main project section, Flexbox for the header and footer, pseudo-classes and pseudo-elements for interactive and decorative styling, and combinators and attribute selectors for structural element targeting.

Notes CSS Grid finally made sense to me with this one. Before this assignment I was using Flexbox for everything and forcing it to do things it wasn't great at. The pseudo-element requirement made me think more carefully about what belongs in the HTML and what's just presentation.
HW4 Module 4

Country Management System

TypeScript DOM Interfaces Classes Type predicates

A TypeScript app using interfaces, classes, inheritance, type assertions, and type predicates to manage a list of country objects. Filters a mixed array of RainyCountry, SnowyCountry, and IslandCountry objects down to snowy ones and renders everything to the DOM with running totals.

Notes TypeScript was new going into this. The interface and class structure felt familiar from other typed languages I've worked with over the years. The type predicate pattern was something I hadn't seen before. Having the compiler catch type errors before runtime was actually useful, not just an academic exercise.
HW5 Module 5

Fresh Basket Grocery — Inventory App

React Vite JSX Components Props JSON data

A React grocery store inventory app called Fresh Basket Grocery. Set up with Vite and organized into Header, Footer, InventoryList, and InventoryItem components. Inventory data in a JSON file, rendered dynamically. Warm amber and cream color palette with a custom logo.

Notes First time building anything real with React. The component model took some adjustment coming from just writing pages. Props made sense once the InventoryItem pattern clicked. Vite was noticeably faster to get going than Create React App the one time I'd tried that before.
HW6 Module 6

Fresh Basket Grocery — Routing Update

React React Router v6 Dynamic routing useParams useNavigate

Extended the HW5 inventory app with react-router-dom v6. Each item on the home page links to a ProductDetail page via a unique SKU-based URL. Added description and image fields to the inventory data, product images in the public folder, and a back navigation link from the detail view.

Notes Dynamic routing with useParams was the main new concept here. Getting the SKU to match correctly between the URL and the data lookup took a few tries. useNavigate for the back button ended up being simpler than I thought it would be. This was probably the point where React started to feel less foreign.

HW5 and HW6 were built as a standalone React/Vite app. The version below runs the compiled build embedded in this site.
